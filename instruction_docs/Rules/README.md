## 🔗 核心业务流

### 文档生命周期
```
上传 → 待解析 → 解析中 → 已解析 → 审核通过/拒绝
                     ↓
              (解析失败) → 错误状态
```

### Q&A 生命周期
```
创建 → 待审核 → 审核通过(发布) / 审核拒绝
                ↓
         (发现重复) → 合并提示
                ↓
         (管理员删除) → 回收站 → 彻底删除/恢复
```

## ⚙️ 业务约束

| 约束类型 | 规则 | 说明 |
|----------|------|------|
| 上传限制 | 文件大小 ≤ 50MB | 可配置 |
| 上传限制 | 文件类型 | .pdf/.docx/.xlsx/.txt/.csv |
| 查重阈值 | 相似度 ≥ 85% | 触发重复警告 |
| 密码策略 | 最少 6 位 | 无复杂度要求 |
| Token 有效期 | 24 小时 | 可配置 |

## 📊 统计规则

### 文档统计
- `total_documents`: 所有非删除文档数
- `doc_parsing_count`: 正在解析的文档数
- `doc_parsed_count`: 已解析完成的文档数
- `documents_by_store`: 按门店分组的文档数

### Q&A 统计
- `total_qa_items`: 所有非删除 Q&A 数
- `published_count`: 已发布的 Q&A 数
- `pending_count`: 待审核的 Q&A 数
- `rejected_count`: 已拒绝的 Q&A 数
- `qa_by_category`: 按分类分组的 Q&A 数

## 🔍 查询规则

### 筛选优先级
1. 状态筛选（优先）
2. 门店筛选
3. 分类筛选
4. 时间范围筛选

### 分页规则
- 默认每页 100 条
- 最大限制 500 条
- 支持跳页查询
