# 项目文件夹说明

> 4S店售后知识库系统 - 目录结构详解

---

## 📁 根目录结构概览

```
📦 门店知识库项目
├── 🔧 开发环境: .trae, .venv, venv, lib    ← 自动生成的，不用管
├── 💻 代码: app                       ← 你的主要工作区
├── ⚙️ 配置: config                    ← 启动和部署相关
├── 📚 文档: instruction_docs          ← 使用说明和开发规范
├── 🛠️ 脚本: scripts                   ← 数据库、部署工具
├── 🧪 测试: tests                     ← 测试代码
├── 📂 上传: uploads                   ← 用户上传的文件
└── 💾 数据: .env, *.db                ← 配置和数据库文件
```

---

## 🔧 开发环境相关

| 文件夹 | 作用 | 说明 |
|--------|------|------|
| **.trae** | IDE 配置 | Trae 编辑器的配置文件和缓存 |
| **.venv** | 虚拟环境 | Python 依赖包存放位置（自动创建）|
| **venv** | 虚拟环境 | 另一个 Python 虚拟环境 |
| **lib** | 依赖库 | 项目使用的第三方库（可能是手动安装的）|

💡 **提示**：这些文件夹是自动生成的，日常开发不需要手动修改。

---

## 💻 核心代码（app/）

这是项目的主要业务代码目录：

```
app/
├── api/                    # API 接口层
│   ├── admin/             # 管理员接口（审核、统计、管理）
│   ├── store/             # 门店接口（上传、查询）
│   └── auth.py            # 认证接口（登录、注册）
│
├── core/                   # 核心配置层
│   ├── config.py          # 环境变量和配置管理
│   ├── database.py        # 数据库连接和会话管理
│   ├── security.py        # JWT 认证、密码加密
│   └── logging_config.py  # 日志配置
│
├── models/                 # 数据模型层（数据库表结构）
│   ├── user.py            # 用户表
│   ├── document.py        # 文档表
│   └── qa_item.py         # Q&A 表
│
├── schemas/                # 数据验证层（Pydantic 模型）
│   ├── user.py            # 用户数据验证
│   ├── document.py        # 文档数据验证
│   └── qa_item.py         # Q&A 数据验证
│
├── services/               # 业务逻辑层
│   ├── auto_classify.py   # AI 自动分类（关键词+LLM）
│   ├── document_parser.py # 文档解析（PDF/Word/Excel）
│   ├── qa_deduplication.py# Q&A 查重（相似度检测）
│   ├── llm_service.py     # LLM 调用服务
│   └── kb_categories.py   # 知识库分类体系定义
│
├── static/                 # 静态资源
│   └── css/               # 样式文件
│
├── templates/              # HTML 模板
│   ├── login.html         # 登录页面
│   ├── admin.html         # 管理后台
│   ├── store.html         # 门店页面
│   └── index.html         # 首页
│
├── __init__.py
└── main.py                 # 应用入口（FastAPI 实例）
```

### 各层职责说明

| 层级 | 职责 | 对应文件夹 |
|------|------|-----------|
| **API 层** | 接收 HTTP 请求，返回响应 | `api/` |
| **核心层** | 配置、数据库、安全、日志 | `core/` |
| **模型层** | 定义数据库表结构 | `models/` |
| **验证层** | 数据校验和序列化 | `schemas/` |
| **业务层** | 实现具体业务功能 | `services/` |
| **展示层** | 网页模板和静态资源 | `templates/`, `static/` |

## 🎯 开发工作流

### 日常开发涉及的主要目录

1. **修改业务逻辑** → `app/services/`
2. **添加新接口** → `app/api/`
3. **修改数据库** → `app/models/` + `scripts/db/`
4. **修改页面** → `app/templates/` + `app/static/`
5. **本地测试** → `tests/`
6. **更新文档** → `instruction_docs/`

### 启动命令

```bash
# 本地开发
python config/run_local.py

# 或使用简化版
python config/run_simple.py

# Windows 双击启动
config/start_local.bat
```

---

## 📌 重要提示

1. **不要手动修改**：`.trae`, `.venv`, `venv`, `lib` 这些自动生成的目录
2. **不要提交 Git**：`.env`, `*.db`, `uploads/` 等包含敏感信息或临时文件的目录
3. **定期备份**：`knowledge_base.db` 是核心数据，建议定期备份
4. **保持同步**：修改代码后记得同步更新 `instruction_docs/` 中的文档
