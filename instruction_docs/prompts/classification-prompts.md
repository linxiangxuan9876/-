# 分类 Prompts

> 用于 Q&A 分类的专用 Prompt 模板

---

## 1. 主分类 Prompt

### 简洁版
```markdown
请为以下 Q&A 选择最合适的主分类：

问题：{question}
答案：{answer}

可选分类：保修政策、保养维护、故障诊断、配件更换、服务预约、门店活动

只返回一个分类名称，不要其他内容。
```

### 详细版
```markdown
## 任务
为以下 Q&A 内容选择一个主分类。

## Q&A 内容
问题：{question}
答案：{answer}

## 分类定义
| 分类 | 定义 | 典型问题示例 |
|------|------|-------------|
| 保修政策 | 保修期限、条件、流程 | 保修期是多久？ |
| 保养维护 | 保养项目、周期、内容 | 多久保养一次？ |
| 故障诊断 | 故障识别、诊断、解决 | 发动机异响怎么办？ |
| 配件更换 | 配件规格、更换周期 | 刹车片多久换？ |
| 服务预约 | 预约流程、取消改期 | 如何预约保养？ |
| 门店活动 | 促销、优惠、会员 | 最近有什么优惠？ |

## 输出要求
- 只选择一个最匹配的分类
- 直接返回分类名称
- 不需要解释
```

## 2. 子分类 Prompt

### Prompt
```markdown
## 任务
为 Q&A 内容生成一个子分类。

## 内容
问题：{question}
主分类：{main_category}

## 要求
- 子分类应简洁（2-4个字）
- 子分类应具体描述问题类型
- 同一主分类下的子分类应互不重复

## 示例
主分类：保修政策
- 子分类示例：保修期限、保修范围、保修流程、免费维修、自费维修

主分类：保养维护
- 子分类示例：定期保养、机油更换、滤芯更换、轮胎保养

主分类：故障诊断
- 子分类示例：发动机故障、刹车故障、电气故障、空调故障

## 输出
直接返回子分类名称，不要其他内容。
```

## 3. 批量分类 Prompt

### Prompt
```markdown
## 任务
批量为以下 Q&A 列表进行分类。

## 输入格式
序号|问题|答案
1|问题1内容|答案1内容
2|问题2内容|答案2内容
...

## 分类体系
【主分类】保修政策、保养维护、故障诊断、配件更换、服务预约、门店活动

【子分类命名规则】
- 简洁：2-4个字
- 具体：描述具体问题类型
- 不重复：同一主分类下子分类互不重复

## 输出要求
返回 JSON 数组格式：
[
  {"index": 1, "main_category": "主分类", "sub_category": "子分类"},
  {"index": 2, "main_category": "主分类", "sub_category": "子分类"}
]

不要包含其他内容。
```

### 批量分类示例

**输入：**
```
1|轮胎保修多久|轮胎保修期为2年或5万公里，以先到者为准
2|怎么预约保养|可通过手机APP或电话预约，选择就近门店
3|发动机故障灯亮|请尽快到店检测，使用诊断仪读取故障码
```

**输出：**
```json
[
  {"index": 1, "main_category": "保修政策", "sub_category": "保修期限"},
  {"index": 2, "main_category": "服务预约", "sub_category": "预约方式"},
  {"index": 3, "main_category": "故障诊断", "sub_category": "警告灯"}
]
```

## 4. 分类修正 Prompt

### Prompt
```markdown
## 任务
检查并修正以下 Q&A 的分类是否正确。

## 内容
问题：{question}
答案：{answer}
当前主分类：{current_main_category}
当前子分类：{current_sub_category}

## 分类定义
请参考以下分类定义，判断当前分类是否准确：

【保修政策】关于车辆保修期限、条件、流程的问题
【保养维护】关于定期保养项目、周期、内容的问题
【故障诊断】关于车辆故障识别、诊断、解决的问题
【配件更换】关于汽车配件规格、更换周期、注意事项
【服务预约】关于预约流程、注意事项、取消改期
【门店活动】关于促销活动、服务优惠、会员权益

## 输出格式
返回 JSON：
{
  "is_correct": true/false,
  "suggested_main_category": "建议的主分类（如果不正确）",
  "suggested_sub_category": "建议的子分类（如果不正确）",
  "reason": "修正理由（如果需要修正）"
}
```

## 5. 分类置信度 Prompt

### Prompt
```markdown
## 任务
评估以下 Q&A 分类的置信度。

## 内容
问题：{question}
答案：{answer}
指定的主分类：{main_category}
指定的子分类：{sub_category}

## 置信度等级
- high（90-100%）：分类明显准确
- medium（70-90%）：分类基本正确，可能有更合适的
- low（50-70%）：分类可能不正确
- very_low（<50%）：分类很可能不正确

## 输出格式
返回 JSON：
{
  "confidence": "high/medium/low/very_low",
  "reason": "评估理由",
  "alternative_category": "更合适的分类（如果有）"
}
```
